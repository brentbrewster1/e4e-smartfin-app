# SmartFin Apple Watch Data Capture + Local Storage Implementation Brief

Goal:
Build the Apple Watch side of the SmartFin research pipeline.

The watch app needs to:
1. Receive raw data packets from the SmartFin device.
2. Parse and validate each packet.
3. Decode the payload based on packet type.
4. Match fin sensor timestamps with Apple Watch GPS timestamps.
5. Save raw packets and parsed readings locally on the watch.
6. Mark records as unsent so they can later be uploaded to the Django REST API.

---

## 1. Raw Packet Format

The SmartFin sends packets in this envelope format:

```text
+---------+---------+--------+-----------+---------...--------+
| version | type    | seq    | payloadLen | payload            |
| 1 byte  | 1 byte  | 2 bytes| 2 bytes    | up to 230 bytes    |
+---------+---------+--------+-----------+---------...--------+
```

Expected byte layout:

```text
byte 0: version
byte 1: type
byte 2-3: sequence number
byte 4-5: payload length
byte 6...: payload
```

Important:
- Confirm endianness with the SmartFin team.
- Until confirmed, implement parsing with a configurable byte order.
- Default assumption: little-endian for UInt16 values.
- Validate that payloadLen equals the number of bytes after the 6-byte header.
- Preserve the raw packet bytes during development for debugging.

## 2. Recommended Watch App Components

Create these Swift components:

```text
SmartFinBLEManager
- Handles BLE connection to SmartFin.
- Subscribes to the packet characteristic.
- Receives raw Data objects from the device.
- Passes raw packet Data to PacketParser.

PacketParser
- Parses the 6-byte packet header.
- Validates version, type, seq, and payloadLen.
- Extracts payload.
- Returns a SmartFinPacket struct.

PayloadDecoder
- Decodes payload based on packet type.
- Converts raw payload into typed sensor readings.
- This will need to be updated once the payload layout is confirmed.

SequenceTracker
- Tracks last sequence number.
- Detects missing, duplicate, or out-of-order packets.
- Adds debug flags to saved packet records.

SensorTimeAligner
- Converts fin timestamps into watch/UTC timestamps.
- Handles offset between fin time and watch time if fin uses relative time.
- Matches sensor readings to the nearest GPS point.

LocationManager
- Uses CoreLocation on watchOS.
- Records GPS samples with timestamp, latitude, longitude, horizontal accuracy, speed if available.
- Provides nearest-GPS lookup for a given sensor timestamp.

LocalDataStore
- Saves raw packets and parsed readings locally.
- Can use SwiftData, Core Data, or SQLite.
- Recommended MVP: SwiftData if the project supports modern watchOS.
- Safer/debuggable alternative: SQLite through GRDB or raw SQLite.

UploadQueue
- Not the main focus yet, but local records should include uploadStatus.
- Records should be saved as unsent by default.
```

## 5. Packet Parser Behavior

Create a `PacketParser` class or struct.

Responsibilities:
1. Reject packets shorter than 6 bytes.
2. Read version from byte 0.
3. Read type from byte 1.
4. Read seq from bytes 2-3.
5. Read payloadLen from bytes 4-5.
6. Extract payload from byte 6 onward.
7. Validate that payload.count == payloadLen.
8. Return `SmartFinPacket`.
9. Throw or return a parse error if invalid.

Pseudo-code:

```swift
func parse(_ data: Data, receivedAt: Date = Date()) throws -> SmartFinPacket {
    guard data.count >= 6 else {
        throw PacketParseError.tooShort
    }

    let version = data[0]
    let type = data[1]

    let seq = readUInt16(data, offset: 2, endianness: .little)
    let payloadLen = readUInt16(data, offset: 4, endianness: .little)

    let payload = data.subdata(in: 6..<data.count)

    guard payload.count == Int(payloadLen) else {
        throw PacketParseError.invalidPayloadLength(
            expected: Int(payloadLen),
            actual: payload.count
        )
    }

    return SmartFinPacket(
        version: version,
        type: type,
        seq: seq,
        payloadLen: payloadLen,
        payload: payload,
        rawData: data,
        receivedAt: receivedAt
    )
}
```

Add an enum:

```swift
enum PacketParseError: Error {
    case tooShort
    case invalidPayloadLength(expected: Int, actual: Int)
    case unsupportedVersion(UInt8)
    case unknownType(UInt8)
}
```

## 6. Sequence Tracking

Create a `SequenceTracker`.

Goal:
Detect packet gaps, duplicates, and out-of-order packets.

Behavior:
- Store the last sequence number received.
- For the first packet, accept it.
- If seq == lastSeq, mark duplicate.
- If seq == lastSeq + 1, normal.
- If seq > lastSeq + 1, mark missing previous packet(s).
- If seq < lastSeq, mark out-of-order unless sequence rollover is expected.
- Account for UInt16 rollover from 65535 back to 0.

Pseudo-code:

```swift
final class SequenceTracker {
    private var lastSeq: UInt16?

    func inspect(seq: UInt16) -> SequenceStatus {
        guard let last = lastSeq else {
            lastSeq = seq
            return .normal
        }

        let expected = last &+ 1

        if seq == last {
            return .duplicate
        }

        if seq == expected {
            lastSeq = seq
            return .normal
        }

        if isSeqAhead(seq, comparedTo: expected) {
            lastSeq = seq
            return .missingPrevious(expected: expected, actual: seq)
        }

        return .outOfOrder(expected: expected, actual: seq)
    }
}
```

## 7. Payload Decoding

The exact payload format is not fully known yet.

Implement the decoder as a switch by packet type, but leave clear TODOs until the SmartFin payload spec is confirmed.

Known API ensemble types from backend:

```text
01 = Temperature + water status only
11 = Full set: temperature, water, 9-axis IMU, GPS
12 = High data rate IMU only
```

Create something like:

```swift
func decode(packet: SmartFinPacket) throws -> DecodedSensorReading {
    switch packet.type {
    case 0x01:
        return try decodeTemperatureWater(packet.payload)

    case 0x11:
        return try decodeFullSet(packet.payload)

    case 0x12:
        return try decodeHighRateIMU(packet.payload)

    default:
        throw PacketParseError.unknownType(packet.type)
    }
}
```

Important TODO:
Ask SmartFin team for:
- Endianness.
- Timestamp format.
- Payload layout per type.
- Temperature units and scaling.
- IMU units and scaling.
- Water status encoding.
- Whether type values are ASCII strings, numeric bytes, or another enum.

## 8. Timestamp Handling

This is the most important part.

### Case A: Fin timestamp is real UTC/Unix time

If the fin timestamp is Unix milliseconds, convert directly:

```swift
let watchTimestamp = Date(timeIntervalSince1970: Double(finTimestampMs) / 1000.0)
```

### Case B: Fin timestamp is relative time since boot/session

If the fin timestamp is milliseconds since fin boot, the watch must estimate a time offset.

When the first valid fin packet arrives:

```text
finTimeAtReceive = packet.finTimestampRaw
watchTimeAtReceive = packet.receivedAt
offset = watchTimeAtReceive - finTimeAtReceive
```

Then:

```text
estimatedWatchTimestamp = offset + finTimestampRaw
```

Store:
- `finTimestampRaw`
- `finTimestampUnit`
- `watchReceivedAt`
- `watchTimestamp`

This makes the dataset auditable.

## 9. GPS Matching

Use nearest-neighbor matching for MVP.

Algorithm:
1. Keep a rolling list of recent GPS samples.
2. When a decoded sensor reading has a watchTimestamp, find the GPS sample with the smallest absolute time difference.
3. Only match if the difference is below a threshold.

Recommended threshold:

```text
maxGpsMatchGap = 2.0 seconds
```

Pseudo-code:

```swift
func nearestGPSSample(to timestamp: Date, maxGapSeconds: TimeInterval = 2.0) -> WatchGPSSample? {
    let closest = gpsSamples.min {
        abs($0.timestamp.timeIntervalSince(timestamp)) <
        abs($1.timestamp.timeIntervalSince(timestamp))
    }

    guard let closest else { return nil }

    let delta = abs(closest.timestamp.timeIntervalSince(timestamp))

    guard delta <= maxGapSeconds else {
        return nil
    }

    return closest
}
```

Store the GPS time delta in milliseconds:

```swift
gpsTimeDeltaMs = gps.timestamp.timeIntervalSince(sensorTimestamp) * 1000.0
```

## 10. BLE Manager Shape

Create a BLE manager using CoreBluetooth.

```swift
final class SmartFinBLEManager: NSObject, ObservableObject {
    @Published var connectionState: ConnectionState = .disconnected

    private var centralManager: CBCentralManager?
    private var peripheral: CBPeripheral?
    private let packetParser: PacketParser
    private let dataStore: LocalDataStore
}
```

Responsibilities:
- Start scanning for SmartFin peripheral.
- Connect to device.
- Discover services.
- Discover characteristics.
- Subscribe to the data characteristic.
- On notification, receive `Data`.
- Parse packet.
- Save raw packet.
- Decode payload.
- Timestamp-align.
- Match GPS.
- Save SensorReading as pending upload.

Pseudo-flow:

```swift
func peripheral(
    _ peripheral: CBPeripheral,
    didUpdateValueFor characteristic: CBCharacteristic,
    error: Error?
) {
    guard error == nil else { return }
    guard let data = characteristic.value else { return }

    let receivedAt = Date()

    do {
        let packet = try packetParser.parse(data, receivedAt: receivedAt)
        let sequenceStatus = sequenceTracker.inspect(seq: packet.seq)

        let rawPacketRecordId = try dataStore.saveRawPacket(
            packet,
            sequenceStatus: sequenceStatus
        )

        let decoded = try payloadDecoder.decode(packet: packet)

        let watchTimestamp = timeAligner.toWatchTime(
            finTimestampRaw: decoded.finTimestampRaw,
            receivedAt: receivedAt
        )

        let gps = locationManager.nearestGPSSample(to: watchTimestamp)

        try dataStore.saveSensorReading(
            decoded,
            rawPacketId: rawPacketRecordId,
            packet: packet,
            watchTimestamp: watchTimestamp,
            gps: gps
        )

    } catch {
        try? dataStore.saveInvalidRawPacket(data, receivedAt: receivedAt, error: error)
    }
}
```

## 11. Location Manager

Use CoreLocation.

Responsibilities:
- Request location authorization.
- Start location updates during an active surf session.
- Save GPS points locally.
- Keep recent samples in memory for fast nearest-neighbor matching.

Pseudo-shape:

```swift
final class WatchLocationManager: NSObject, ObservableObject, CLLocationManagerDelegate {
    private let manager = CLLocationManager()
    private let dataStore: LocalDataStore
    private var recentSamples: [WatchGPSSample] = []

    func startSessionLocationUpdates(sessionId: UUID) {
        manager.requestWhenInUseAuthorization()
        manager.desiredAccuracy = kCLLocationAccuracyBest
        manager.distanceFilter = kCLDistanceFilterNone
        manager.startUpdatingLocation()
    }

    func stopSessionLocationUpdates() {
        manager.stopUpdatingLocation()
    }

    func locationManager(_ manager: CLLocationManager, didUpdateLocations locations: [CLLocation]) {
        for location in locations {
            let sample = WatchGPSSample(
                timestamp: location.timestamp,
                latitude: location.coordinate.latitude,
                longitude: location.coordinate.longitude,
                horizontalAccuracy: location.horizontalAccuracy,
                altitude: location.altitude,
                speed: location.speed,
                course: location.course
            )

            recentSamples.append(sample)
            trimRecentSamples()
            try? dataStore.saveGPSSample(sample)
        }
    }
}
```

## 12. Save Locally First

Every valid reading should be saved locally before upload.

This is important because:
- The watch may have no cellular connection.
- The user may be surfing away from the iPhone.
- Upload may fail.
- Research data should not be lost.

Each `SensorReading` starts with:

```text
uploadStatus = pending
uploadedAt = nil
```

Later the uploader can query:

```text
all readings where uploadStatus == pending
```

and send them to the backend.

## 18. Important Implementation Principle

For this research project, do not throw away raw data.

Always save:

```text
raw packet bytes
parsed header
receivedAt timestamp
sequence number
parse status
decoded values
matched GPS point
upload status
```

This makes the data traceable from:

```text
SmartFin packet → watch reading → uploaded backend record → frontend/research analysis
```
